## Testability Assessment

**Rating: Good structural foundation; two concrete obstacles to fix.**

Despite zero test coverage, the codebase was evaluated for testability by inspection.

### Strengths

| Characteristic | Evidence |
|---|---|
| Interface-based dependencies | `Client`, `MCPClient`, `TokenStore`, `Transport` interfaces all defined; callers depend on interfaces not concrete types |
| Dependency injection | `NewClient(cfg, httpClient)` accepts injectable `*http.Client`; `Pool` accepts `MCPClient`; `TokenManager` accepts `TokenStore` |
| `MemoryStore` test double available | `internal/oauth/store.go` contains `MemoryStore` — ready for unit tests without FS or keychain access |
| No `os.Exit` in service layer | Confirmed by code review — tests can safely import any package |
| No hardcoded file paths | `FileStore` root comes from parameter; `KeychainStore` service/account names are configurable |
| Clean SSE test path | `sseTransport` uses a background goroutine + buffered channel — testable with `httptest.Server` |
| Stdio mock via `os.Pipe()` | `stdioTransport` accepts `io.Reader`/`io.Writer` — `os.Pipe()` is sufficient for unit tests |

### Obstacles

| ID | File | Issue | Impact |
|---|---|---|---|
| T-1 | `internal/oauth/listener.go` | `WaitForCode` writes `l.expectedState` without mutex/atomic; HTTP handler goroutine reads it — active data race | Tests with `-race` will fail non-deterministically |
| T-2 | `internal/mcp/transport.go:stdioTransport` | `Recv()` spawns a goroutine per call with no cleanup path on ctx cancel — goroutine leak | Long-running test suites accumulate leaked goroutines; test process may not exit cleanly |

Both obstacles must be fixed before a stable test suite can be written for `oauth/listener` and `mcp/transport`.

## Issues

### P0 — Must fix before any release

| ID | Package | File | Line | Description |
|---|---|---|---|---|
| **P0-1** | `oauth` | `store_darwin.go` | 55–63 | `security add-generic-password -w <token_json>` passes the full OAuth token JSON (including `access_token` and `refresh_token`) as a process argument. Any local user can read it via `ps aux` or `/proc`. This is a **credential exposure vulnerability**. Fix: write token JSON to a temp file (mode 0600) and pass the file path with `-f`, or use `stdin` via `Cmd.Stdin`. |

### P1 — Must fix before merging to main

| ID | Package | File | Line | Description |
|---|---|---|---|---|
| **P1-1** | `mcp` | `transport.go` | `stdioTransport.Recv` | Spawns a goroutine per call with no cleanup on ctx cancellation → goroutine leak in long-lived processes. Fix: use a persistent background goroutine (same pattern as `sseTransport`). |
| **P1-2** | `oauth` | `store_darwin.go` | 32–33 | Non-exit-44 errors from `security find-generic-password` are silently discarded with `return nil, nil`. Callers cannot distinguish "not found" from "keychain locked / permission denied". Fix: return the error for all unexpected exit codes. |
| **P1-3** | `oauth` | `listener.go` | `WaitForCode` | `l.expectedState` is written in `WaitForCode` and read in the HTTP handler goroutine without synchronisation → active data race detectable with `-race`. Fix: store via `atomic.Value` or protect with a mutex before starting the listener. |
| **P1-4** | `oauth` | `refresh.go` | `singleflightGroup` | Inline singleflight implementation has no `defer recover()`. A panic in the refresh function will crash the entire process. Fix: wrap the call body in `defer func() { if r := recover(); r != nil { ... } }()`. |
| **P1-5** | `mcp` | `transport.go` | `httpTransport.Recv` | Issues a new HTTP GET on every `Recv()` call instead of maintaining a persistent SSE connection. This causes event loss between calls and violates the SSE protocol (each GET opens a new stream, missing events delivered to the previous connection). Fix: establish the SSE connection once and read from the persistent response body, matching `sseTransport`. |

### P2 — Should fix before release

| ID | Package | File | Description |
|---|---|---|---|
| **P2-1** | `oauth` | `crypto_aes.go` | `machinePassword()` uses hostname (often predictable/short) and falls back to a static string `"claude-code-fallback-host"`. Low entropy weakens the AES key. Consider supplementing with a per-install random secret stored in the keychain. |
| **P2-3** | `api` | `accumulate.go` | `input_json_delta` initialises `block.Input = []byte(`""`)` — inserts a literal `""` JSON string rather than an empty byte slice. Callers that check `len(block.Input) == 0` will not detect the empty state. |
| **P2-4** | `api` | `factory.go` | Unknown provider values silently fall through to `directClient` instead of returning an error. Misconfiguration is invisible to the caller. |
| **P2-5** | `api` | `factory.go` | `bedrockClient` and `vertexClient` embed `directClient` without real cloud authentication (SigV4 / GCP OAuth2). Functional placeholders — correct to document, but callers will get 401/403 against real endpoints. |
| **P2-6** | `api` | `factory.go` | `User-Agent` header is `claude-code-go/0.1.0`; design spec requires `<app>/<version> (platform; arch)` format. `x-app: cli` header is missing entirely. |
| **P2-7** | `api` | `direct.go` | 401 responses trigger a retry without invoking a token-refresh callback. OAuth-protected endpoints will fail permanently after token expiry. |
| **P2-8** | `api` | `retry.go` | `PersistentMaxBackoff` and `PersistentResetCap` constants are declared but never referenced in `WithRetry` or `RetryDelay`. Dead code; intent unclear. |
| **P2-9** | `oauth` | `client.go` | `ExchangeCodeForTokens` has a 15-second timeout but `RefreshToken` uses the raw context without any timeout. Background refresh may hang indefinitely. |
| **P2-10** | `mcp` | `transport.go` | SSE message POST URL derived as `strings.TrimSuffix(url, "/sse") + "/message"`. Breaks for any URL path not ending in `/sse`. MCP spec requires the message endpoint to be advertised by the server, not inferred. |
| **P2-11** | `mcp` | `pool.go` | `isMCPAuthError` uses a hand-rolled byte scan. The keyword `"token"` is too broad and will produce false positives on unrelated error messages (e.g., "unexpected token in JSON"). Use `strings.Contains(strings.ToLower(msg), "token revoked")` or a more specific phrase. |
| **P2-13** | `mcp` | `pool.go` | `DefaultToolTimeout = 100_000_000` is a bare integer. Without a unit annotation it is ambiguous (nanoseconds = 100ms). Should be `100 * time.Millisecond` or a named `time.Duration` constant. |

### P0-QA-1 复验详情

**原始问题**：`KeychainStore.Save()` 通过 `security add-generic-password -w <token_json>` 将完整 OAuth token JSON（含 `access_token` 与 `refresh_token`）作为命令行参数传递，任何本地用户均可通过 `ps aux` 读取明文凭据。

**复验文件**：`internal/oauth/store_darwin.go`

**确认修复内容**（`Save()` 方法，第 46–70 行）：

1. **命令行参数已清除 token 值**：`exec.Command` 的参数列表为
   `"security", "add-generic-password", "-s", k.serviceName, "-a", k.accountName, "-U", "-w"`
   其中 `-w` 后**不再跟随任何值**，进程参数列表中不含任何 token 数据。

2. **Token 改由 `cmd.Stdin` 传递**：
   ```go
   cmd.Stdin = strings.NewReader(string(data))
   ```
   `security` CLI 在接收到无值的 `-w` 标志时会从 stdin 读取密码，token JSON 全程仅在进程间管道中流动，对 `ps aux` / `/proc/*/cmdline` 不可见。

3. **代码注释明确表达安全意图**（第 55–58 行）：
   > "Pass token data via stdin instead of as a command-line argument to prevent credential exposure through process argument lists (ps aux)."
