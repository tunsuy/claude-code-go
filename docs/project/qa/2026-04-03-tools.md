## 3. 单元测试质量评估

### 3.1 优点

**`fileops/glob_test.go`（16 个测试函数）**：
- 使用 `package fileops_test` 黑盒模式，符合 test-strategy.md 要求。
- 使用 `t.TempDir()` 进行文件系统隔离，无全局状态污染。
- 覆盖了 `IsConcurrencySafe`、`IsReadOnly`、`ValidateInput`、`UserFacingName`、`MapResultToToolResultBlock`、`InputSchema`、接口合规性 等多个维度。
- `makeTree` 辅助函数封装良好，测试体简洁。
- 边界用例（无匹配、隐藏目录跳过、不存在路径）有专项测试。

**`fileops/grep_test.go`（18 个测试函数）**：
- 同样使用黑盒模式和 `t.TempDir()`。
- 覆盖三种输出模式（content / files_with_matches / count）、Include 过滤器、单文件搜索、正则捕获。
- 验证了行号（`Line` 字段）精确性，属于有意义的功能回归检测。
- `makeTextTree` 与 `grepCall` 辅助函数复用良好。

### 3.2 不足

| 问题 | 位置 | 影响 |
|------|------|------|
| 部分测试断言不完整 | `grep_test.go` `TestGrepTool_FilesWithMatchesMode`：只检查 `len(out.Matches)==0`，未验证 `out.Files` 中具体文件路径 | 可能漏报 regression |
| 缺少 `Aliases` 测试 | `glob_test.go` | `GlobTool.Aliases()` 零覆盖 |
| 缺少 `SearchHint` 测试 | 两个测试文件均未覆盖 | 轻微 |
| 无表驱动测试 | `ValidateInput` 的多个 bad-input case 可合并为表驱动测试 | 不符合 test-strategy.md 推荐风格 |
| 无 `context.WithCancel` 取消路径测试 | `grep_test.go`、`glob_test.go` | 无法验证上下文取消行为 |
| `MapResultToToolResultBlock` 分支覆盖不完整 | `grep_test.go`：未测试 `files_with_matches`/`count` 模式下的格式化输出 | `grep.go:309` 覆盖 61.1% |

## 4. 可测试性评估

### 4.1 良好实践（已体现）

- `fileops` 包中所有工具均以**单例导出变量**（`GlobTool`、`GrepTool` 等）暴露，可直接在测试中调用，无需 mock 复杂依赖。
- `Call` 方法参数接受 `*tool.UseContext`，可传 `nil` 简化测试，或注入带 `context.Context` 的实例以测试超时/取消。
- `writeFileAtomic` 函数已独立提取，具备独立测试条件。
- `expandPath` 对 `~` 展开逻辑已独立，可单测。

### 4.2 反模式 / 可测试性缺陷

| 问题 | 位置 | 严重度 | 说明 |
|------|------|--------|------|
| 全局 `defaultHTTPClient` 无法注入 | `web/webfetch.go:69`、`web/websearch.go` | P1 | `webFetchTool` 和 `webSearchTool` 直接使用包级变量，测试必须真实发起 HTTP 请求，无法 mock，无法单测。 |
| `os.Getenv("BRAVE_API_KEY")` 在 `Call` 内直接读取 | `web/websearch.go:122` | P1 | 未使用 `t.Setenv()` 可注入的方式，强制依赖真实环境变量。设计上应作为构造函数参数或 `UseContext` 字段传入。 |
| 全局 `fetchCache` 无清除机制 | `web/webfetch.go:48` | P2 | 包级缓存在并发测试或多轮测试间可能污染状态，缺少 `clearCache()` 测试辅助函数。 |
| `TODO(dep)` stub 全部无法单测 | `agent`, `tasks`, `interact`, `misc` 部分 | P1 | 所有 stub 工具的 `Call` 方法直接返回占位错误，但**连 stub 的 schema / identity 方法都缺少测试**，测试接入点完全空白。 |
| `shell/bash.go` 无接口抽象 | `shell/bash.go` | P2 | `exec.CommandContext` 直接调用，无 `CommandRunner` 接口注入，无法在 CI 无 shell 环境中单测。 |
